Game Of Life:

Game Of Life is a cellular automaton, devised by the British mathematician John Horton Conway in 1970.
The game has a game board of X by Y dead or living cells. The state of each cell depends on its eight neighbors.
Each round will be the state of all cells (depending on the used rule set) calculated. A round is called a generation.

Rules Of Life:

1. Any living cell with less than two living neighbors become lonely and dies caused by underpopulation.
2. Any living cell with two or three living neighbors stays alive.
3. Any living cell with more than three living neighbors starves and dies caused by overpopulation.
4. Any dead cell with exactly three living neighbours gets born caused by optimal conditions.

Views:

General

The whole application is divided into several views.
They can be activated by shortcuts and be hidden by 'ESC'.

Game View

The Game View is the main view and shown after startup.
It contains the field with the cells and the generation counter which displays the current generation. To navigation

Preferences View

The Preferences View can be shown by the shortcut 'P'. It contains the options to adjust the application.
Here are many possibilities from customizing the colors to creating own rules.
An explanation of the preferences can be found hier.

Help View

The Help View is a extension at the left border of the Game View which can be displayed by the shortcut 'H'.
Here are all necessary information for handling with the application.

Tools View

The Tools View is a little bar of the Game View on the upper side.
It contains tools for saving/opening, generating, navigation and drawing as well as some information like the current size of the field.
The view can be activated with 'T' if you are in the Game View. It will be hidden in Help View and Preferences View.

Navigation:

The movement on the game board is with several shortcuts possible. There are the possibilities to move slow (arrow keys/mousewheel), to move fast ([CTRL]+arrow keys) and to zoom ([CTRL]+[+/-]/mousewheel). The position can be resetted by the shortcut 'M' or with the button 'Reset Position' which can be found in the Tools View.

Playing:

There are two options to generate the next generation.
The first option is the autogenerating which generates automatically the next generation with the set delay between. The max. speed will be synchronized with the update rate of the screen.
This mode can be started and stopped by the shortcut 'R'.
The other option is the stepping which allows the calculation of multiple generations in the background at a time. The number of generations per step can be set in the input field in the Tools View. Because of the missing synchronization it is possible, to generate fast many generations.
The stepping can be entered by the shortcut 'Spacebar' which starts the calculation of one step. It can be interrupted by pressing again the 'Spacebar' or by clicking on any cell.

The application supports as another feature the possibility to create own rule sets, so you can set how many neighbors a cell requires to survive or to be born.
The default rules can be formulated that a cell requires 2 or 3 neighbors to survive and 3 neighbors to be born. In all other cases the cell will die.

It is also possible to set the behavior on borders. There are the options that the neighbors of the border cells are on the always other side located or that the bord is surrounded by dead cells or that the board is surrounded by living cells.

Preferences:

Game

Width:
This Option defines the size of the board on the x-axis.

Height:
This Option defines the size of the board on the y-axis.

Behavior On Borders:
The selection here determines the neighbors of the cells on the border (see also here).

Fill New Game With Random Cells:
If this option is active, a new game (shortcut 'N') will be filled with random generated cells.

Relation Of Random Generated Cells:
With this option it is possible to determine the relation of random generated cells. (The option is only available if "Fill New Game With Random Cells" is active)

Autogenerating Delay:
The here set delay is the time which elapses between two generations while autogenerating (shortcut 'R').

Number Of Neighbors Which Are Needed To Survive:
With this option you can set own rule sets. Each button represents one amount of neighbors with which a cell can survive.

Number Of Neighbors Which Are Needed To Be Born:
With this option you can set own rule sets. Each button represents one amount of neighbors with which a cell can be born.

Lock Cells After Initial Generation:
If this option is active, cells can not be modified from generation 1. It is only possible to modify cells in generation 0.

Appearance

Show Grid:
This options allows the possibility to hide the grid around the cells.

Show Generation Counter:
With this option it is possible to hide the generation counter.

Size Of Generation Counter:
This option determines the size of the generation counter. (The option is only available if "Show Generation Counter" is active)

Application

Number Of Threads Used For Calculation:
The option allows to set the number of for calculation used threads. The max. number is usually equala to the number of cores.

Start In Fullscreen Mode:
This option enables the start in fullscreen mode. Fullscreen mode can be left by the shortcut 'F'.

Show Exit Warning:
If this option is inactive, there will be no request to save the current game when leaving the application.

Default Path Of Saved Games:
The default path of games which are saved by pressing 'S' can be set here. If the input field is white, the user doesn't have the permission to write.

Show Saved Game Directory:
With this button it is possible to show the directory where saved games are stored by default in the default file manager.

Color

Color Of Dead Cells:
The here chosen color determines the color of currently dead cells which will stay dead next generation.

Color Of Alive Cells:
The here chosen color determines the color of currently living cells which will stay alive next generation.

Color Of Reviving Cells:
The here chosen color determines the color of currently dead cells which will be born next generation.

Color Of Dying Cells:
The here chosen color determines the color of currently living cells which will die next generation.

Color Of Background:
The here chosen color determines the color of the background outside of the board.

Additional

Generations Per Step:
This option allows to set the generations which are calculated per step. (The option can be found in tools view)

Swap Mouse Actions:
With this option it is possible to swap the "Reviving" and "Killing" action between left and right mouse button. (The option can be found in tools view)
